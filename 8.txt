Experiment 8 — Intrusion Detection and Prevention System using Snort

Aim
To configure and test Snort IDS/IPS and create custom Snort rules.

Step 1: Start Machines
- Start Ubuntu VM
- Start Kali Linux VM

Step 2: Install Snort in Ubuntu
sudo apt-get install snort

Step 3: Find Network Interface
/sbin/route -n

Select correct interface during configuration.

Step 4: Go to Snort Directory
cd /etc/snort

Step 5: Open Configuration File
sudo vi snort.conf

Step 6: Go to Rules Folder
cd rules

Step 7: Open Rule Files
vi local.rules
vi icmp.rules

Step 8: Test Snort Configuration
sudo snort -T -c /etc/snort/snort.conf

Observation:
- Configuration test successful.

Step 9: Start Snort IDS
sudo snort -A console -c /etc/snort/snort.conf

Observation:
- Snort starts monitoring packets.

Step 10: Scan Ubuntu from Kali
nmap 192.168.111.133

Observation:
- Snort detects reconnaissance scan packets.

Step 11: Create Custom Snort Rules
cd /etc/snort/rules
vi local.rules

Add Rules:

alert icmp $EXTERNAL_NET any -> $HOME_NET any (msg:"Shubha"; sid:5889; rev:1;)

alert tcp any any -> $HOME_NET 21 (msg:"FTP attempted"; sid:60001; rev:1;)

alert tcp any any -> $HOME_NET 22 (msg:"SSH attempted"; sid:600022; rev:1;)

Step 12: Verify Configuration Again
sudo snort -T -c /etc/snort/snort.conf

Step 13: Start Snort Again
sudo snort -A console -c /etc/snort/snort.conf

Step 14: Test ICMP Rule
ping 192.168.111.133

Observation:
- “Shubha” ICMP alert detected in Ubuntu.

Step 15: Test FTP Rule
ftp 192.168.111.133

Observation:
- “FTP attempted” alert displayed.

Step 16: Test SSH Rule
ssh ubuntu@192.168.111.133

Observation:
- “SSH attempted” alert displayed.

Result
Successfully configured Snort IDS/IPS, created custom rules, and detected ICMP, FTP, and SSH attacks.
