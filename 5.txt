Experiment 5 — XSS & CSRF using DVWA
and Burp Suite
Aim
To perform Cross Site Scripting (XSS) and Cross Site Request Forgery (CSRF) attacks using
DVWA and Burp Suite.
Theory
Cross Site Scripting (XSS)
Cross Site Scripting is a web security vulnerability in which malicious scripts are injected into web
pages and executed on the client side. It allows attackers to steal cookies, session IDs, and perform
malicious actions in a victim’s browser.
Types of XSS
1. Stored XSS
The malicious script is permanently stored in the web application database.
2. Reflected XSS
The malicious script is reflected immediately from the server response.
Cross Site Request Forgery (CSRF)
CSRF is a web attack in which an attacker tricks an authenticated user into performing unwanted
actions on a web application without their knowledge.

• Kali Linux VM
• Metasploitable2 VM
• DVWA
• Burp Suite
• Firefox Browser

Burp Suite
Select:
Temporary Project

Step 12: Configure Proxy
Go to:
Proxy → Intercept

Intercept ON
Configure Firefox proxy:
HTTP Proxy: 127.0.0.1
Port: 8080
Step 13: Open CSRF Page
In DVWA navigate to:
CSRF
Enter:
New Password
Confirm Password
Step 14: Capture Request
Click:
Change
Burp Suite captures the HTTP request.
Step 15: Send to Repeater
Right click request and select:
Send to Repeater
Click:
Forward

• Password changes successfully.

Change password values and click:
Send
